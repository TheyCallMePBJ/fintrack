## ✏️ Modified Files (10 files)

### Backend — Java

#### `Goal.java` *(was empty)*
- Fully implemented as a JPA database entity
- Fields: `name`, `targetAmount`, `currentSaved`, `targetDate`, `createdDate`
- Added helper methods: `getProgressPercent()`, `getDaysLeft()`, `getRemaining()`
- Added `@Table(name = "app_user")` → fixes H2 reserved keyword bug

#### `GoalService.java` *(was empty)*
- Fully implemented with full CRUD
- `deposit()` — adds savings to a goal
- `getPrediction()` — calculates if you'll hit your goal based on your daily savings rate

#### `TransactionRepository.java`
- Added `findByTypeAndDateBetweenOrderByDateAsc()` — for behaviour analysis
- Added `findByTypeOrderByDateDesc()` — for prediction engine

#### `TransactionService.java`
Added two entire new feature blocks:

**Behaviour Analyser methods:**
- `getBehaviorInsights()` — detects 5 patterns (weekend spending, top category, frequency trend, biggest expense, recurring habits)
- `getWeekendVsWeekday()` — returns weekend vs weekday totals for chart
- `getCategoryTotals()` — returns per-category spending for chart

**Prediction Engine methods:**
- `getPredictedNextMonthExpense()` — weighted 3-month average (50% / 30% / 20%)
- `getExpenseTrendPercent()` — % change vs last month
- `getMonthlyExpenseTotals()` — last N months for sparkline
- `getMonthExpense()` — helper for a specific month

#### `MainController.java`
- Dashboard now passes `predictedExpense`, `expenseTrendPct`, `sparkline` to the template

#### `TransactionController.java`
- Injected `NlpParserService`
- Added `POST /expense/nlp` endpoint — parses natural language and pre-fills the expense form

#### `User.java`
- Added `@Table(name = "app_user")` — fixes H2 crash (`user` is a reserved word in H2)
